LAB 1: LEARN TO GIT

Wat is GIT en hoe moeten we er mee werken?
Voor deze opdracht was het de bedoeling om met 4 personen samen te werken en een repository te delen op Github. Hoofddoel van deze opdracht was het leren werken met de terminal en git. Wanneer we allemaal 1 receptje geschreven hadden was het de bedoeling om een pull request te sturen met onze naam en voornaam + de link van onze taak.

Begrippen GIT in cmd line:

git clone: Een kopie creëeren van je repository op je laptop op een plaats naar keuze.
git status: Geeft de status weer van de huidige rep.
git pull: Je haalt hiermee de laatste versie van je repository binnen.
git add: Als je nieuwe of gewijzigde bestanden toevoegen.
git add -A: De -A staat voor alles, maw alle files.
git commit -m "message": Je gaat hier committen, dit is de tussenstap van add en push. -m en de dubbele quotes duiden aan dat je hier een message moet meegeven met je commit. 
git push: De laatste stap is het pushen. Al je committed bestanden push je door naar jouw repository op GitHub.

Branches

Met branches bedoelen we eigenlijk "takken maken". De master branch is de basis of de standaard branch. Als je bijvoorbeeld de branch 'thais' aanmaakt, dan gaan alle bewerkingen in je files niets veranderen in de master branch. Wanneer je klaar bent met werken op de 'thais' branch kan je deze mergen met de master branch om daarna alles door te pushen naar de repository.

git branch <branchname>: Branch aanmaken met naam naar keuze.
git checkout <branchname>: Switchen van branch.
git checkout -b <branchname>: Nieuwe branch aanmaken en direct over switchen naar deze nieuwe branch.
git merge <branchname>: Aangemaakte branch mergen met de master branch.
git branch -d <branchname>: Verwijderen van aangemaakte branch.

Transitions

"Overgang van iets naar iets zonder tussenstappen", zo werden transitions omschreven tijdens de les. Het is maw een overgang zoals een hover-effect maar minder plotseling, met een effect gespreid over een bepaalde tijd.

Om een transition toe te passen , moet u twee dingen opgeven : . de CSS eigenschap waar u een effect aan wilt toevoegen . de duur van het effect

Transform

De CSS3 transform eigenschap gaat fysiek het uiterlijk van een element veranderen naar de transform vorm die jij gekozen hebt. Het geselecteerde element behoudt zijn oorspronkelijke positie en grootte welke transformatie ook wordt uitgevoerd.

Enkele voorbeelden van 2D transformatie vormen zijn:
. scale() : Element vergroten.
. rotate() : Roteert een element onder een bepaalde hoek.
. skew() : Schuintrekken van een element.
. translate() : Een element verplaatsen.

Om een element in 3D te transformeren moet je de transform-function perspective gebruiken.

Enkele voorbeelden: . rotateX() - rotateY() - rotate3d(x, y, z) : Deze transformatie vormen zorgen er voor dat een element rond een bepaalde as draait.
. perspective() : Perspective voegt een perspectief en een diepte toe aan de 3D-transformaties.

Animations

Een animatie laat een element geleidelijk veranderen van de ene stijl naar de andere, zoveel je maar wil. Dit gebeurd door gebruik te maken van keyframes. Animaties starten ook automatisch en kunnen oneindig loopen.

Enkele eigenschappen:
. animation-name: De naam van de animatie.
. animation-duration: De duur van de animatie.
. animation-timing-function: Hoe de animatie verloopt (ease-in, linear, ease-out,...).
. animation-delay: De animatie kan starten met een vertraging.
. animation-iteration-count: Hoe vaak de animatie wordt afgespeeld.
. animation-direction: De animatie voor- of achterwaarts afspelen.

Voor deze opdracht moesten we een todo-app uitwerken in een eigen mini-framework. De to-do items zullen gewone list-items zijn in een unordered list, maar de bedoeling is dat we deze als 'done' kunnen aanduiden met Javascript zonder het gebruik van jQuery. Wat heb ik geleerd?
. Don't repeat yourself! = DRY
. Performance wordt beter doordat je niet heel de jQuery library moet inladen: laadtijd verminderd
. Functie's zijn niet flexibel genoeg voor grotere apps
. Zo goed als alles in javascript erft over van Object()
. Gebruikt prototypes (voor DRY)

LAB 4: API

Als opdracht moesten we een weerapplicatie maken met open data. De app zelf moest het weer weergeven op je huidige locatie. We moesten de forecast API gebruiken om het weer te weten te komen en HTML5 geolocation voor de huidige locatie.

Functies die we gebruikt hebben tijdens de les:

API

API staat voor Application Programming Interface en is een "bibliotheek" waar publieke data inzit die iedereen kan opvragen met een applicatie.

JSON

Json betekent JavaScript Object Notation, het is makkelijker en overzichtelijker waardoor je goed kan bundelen. Dit maakt Json makkelijker dan XML en sneller met js.

JSONP

JSON met padding, wordt gebruikt voor callbacks.

HTML5 Geolocation

Hiermee kan je de geografische positie (coördinaten) van een gebruiker opvragen en weergeven. Deze coördinaten konden we dan gebruiken in combinatie met de forecast API.

AJAX

Dit is asynchronous javascript en XML / JSON. Dit zorgt ervoor dat je de pagina niet moet refreshen om nieuwe data op te halen. Assymchroom in de definitie wilt zeggen dat het niet gelijktijdig moet. AJAXcall = XHRcall -> ik doe een verzoek nr de server met XHTML.

Andere:

. localstorage (zoals een cookie maar met meer dataopslag)
. fastclick.js (verwijdert 300ms delay op mobiele toestellen + dubbele clicks vermijden)
. use strict (cleanere code schrijven)

LAB 5: NODE.JS EN MONGODB

Gastles door District01 en gewone les. Doordat Node.JS een heel moeilijk onderdeel bleek te zijn, werden hier 2 lessen over gegeven. In de 2e les kregen we de opdracht om een applicatie te maken waarbij meerdere gebruikers zijn/haar naam moesten ingeven en daarbij een korte vraag konden stellen. Deze data moest direct verstuurd worden naar onze database, MongoDB. Via een live-update werden de vragen met naam voor iedereen tegelijkertijd op zijn/haar scherm getoond. Dit doen we met websockets. Hiervoor gebruikten we socket.io.

Maar wat is nu Node.JS?

Node.js is een server-side JavaScript gebouwd op google's V8 JavaScript-Engine, wat we moet PHP kunnen doen we hier met Javascript. Het wordt vooral gebruikt voor apps die hun data live moeten updaten.

Waarom zouden we het gebruiken?

. Javascript
. Asynchrone werkwijze
. Single threaded
. Modulair
. Grote community

Wat is MongoDB?

MongoDB is een databanksysteem dat gebruikt wordt op hoog niveau, het is een flexibel datamodel en is document georiënteerd. Dit databanksysteem is NoSQL. Dit wil zeggen dat je het op meerdere servers kan draaien en het maakt gebruik van simpele queries. MongoDB maakt gebruik van het rest principe : Get, Post, Put, Delete.

Hoe beginnen we hieraan?

Om Node.js en MongoDB te kunnen draaien, gebruiken we voor alles onze Terminal.

Installeer eerst node.js via de website.
https://nodejs.org/

Brew is alleen voor de OSX gebruikers, Windows gebruikers installeren dit op een andere manier. Om de databank te tonen, download je het programma Robomongo.

Twee commando's moeten worden geschreven via onze cmd line om mongoDB te starten:

Om je applicatie automatisch te laten updaten als er iets wordt gewijzigd in één van de bestanden, installeer je nodemon. Alle veranderingen worden gedefinieerd in de terminal.

LAB 6: ANGULAR.JS

Gastles door District01.

Angular.JS is een js framework in samenwerking met Google dat volledig clientside werkt. Dit wil zeggen dat je geen server meer nodig hebt, alle berekeningen worden via de browser gemaakt. Dit framework werkt volgens het MVC principe. MVC staat voor models (class) - views (weergave) - controllers (via methods in). Verder heeft Angular geen dependencies nodig. Je hebt dus geen JQuery nodig om het te laten werken.

Hoe Angular.js gebruiken? Enkele voorbeelden:

Ng-app declareert het root-element (het hoogste element in de structuur) van een Angular-applicatie = HTML.

<!doctype html>
<html ng-app="app">

Ng-bind gaat de inhoud van een DOM-element linken aan de waarde van een expressie.

<input type="text" placeholder="firstname" ng-model="firstname">
<h2>Welcome <span ng-bind="firstname"></span></h2>

Bij de ng-controller wordt een JavaScript-controller-klasse gekoppeld aan de view. Controllers voorzien de scope van functies en waarden.

<div ng-controller="weerController"></div>

angular.module("app").controller("weerController", weerController);

LAB 7: SASS

Sass is een extensie van css en maakt gebruik van variabelen, nesting, mixins, inheritance en operators om de stylesheet zo ordelijk mogelijk te maken. Het gevolg hiervan is dat je minder code moet schrijven en je stylesheet wordt ook veel overzichtelijker.

Sass installeren

Om Sass te installeren moeten we eerst onze command prompt of terminal openen en dit commando schrijven:

gem install sass

//gebruik dit als je een error message krijgt
sudo gem install sass

//check of je sass zeker geïnstalleerd hebt door de versie op te vragen
sass -v

Mappenstructuur

/css
    style.css
/scss
    /partials
        _nav.scss
        _footer.scss
style.scss
_base.scss

Compileren

Ga met de cmd line naar je /scss map en geef onderstaand commando in om van een scss file naar een css file te compileren:

sass style.scss ../css/style.css

En omgekeerd, css naar scss:

sass --watch style.scss:../css/style.css

Variabelen

Door middel van variabelen kan je eigenschappen opslaan die je wil hergebruiken. Bijvoorbeeld kleuren.

$primaire-kleur: #adff2f ;

body {
    color: $primaire-kleur ;
}

Nesting

Met Sass kan je CSS selectors nesten op dezelfde manier als HTML. Voorbeeld:

nav {
    ul {
        margin: 0px;
    }
            li { 
                    display: inline-block; 
            }
    a {
        text-decoration: none;
                    color: #adff2f;
        display: block;
                    padding: 25px;
    }
}

Mixins

Met mixins kan je een groep van CSS declaraties maken die je kan hergebruiken door heel je site. Je kan er zelfs een variabele in doorgeven wat meer flexibiliteit geeft aan je mixin. Hieronder een voorbeeld:

@mixin border-radius ($radius) {
    -webkit-border-radius: $radius ;
    -moz-border-radius: $radius ;
    -ms-border-radius: $radius ;
    border-radius: $radius ;
}

.box { 
            @include border-radius (10px); 
    }

Inheritance

Met de @extend functie kan je een set van CSS eigenschappen meegeven van een selector naar een andere. Dit is het perfecte voorbeeld van DRY (Don't Repeat Yourself).

.message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  @extend .message;
  border-color: green;
}

.error {
  @extend .message;
  border-color: red;
}

.warning {
  @extend .message;
  border-color: yellow;
}

Nog een tip!

Chrome -> settings -> enable CSS source maps

LAB 8: GULP.JS

Gulp is een handige taskmanager om javascript, css bestanden en images makkelijk te compressen. Normaal zou je dit allemaal handmatig moeten doen, maar Gulp doet dit automatisch.

Gulp installeren

Via de commandline:

npm install gulp -g

Daarna ga je gulp lokaal in je project dev dependencies installeren (hiervoor moet je een package.json bestand aanmaken). Deze commando's moeten ook in de juiste map getypt worden (dus eerst met cd naar je projectmap gaan).

//package.json file aanmaken in je projectmap
npm init

//lokaal installleren
npm install gulp --save-dev
